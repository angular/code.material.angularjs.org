The checkbox directive is used like the normal angular checkbox.

As per the Material Design spec the checkbox is in the accent color by default. The primary color palette may be used with the md-primary class.

Usage

Finished ? No Ink Effects Disabled

Attributes

Parameter Type Description
* ng-model expression expression

Assignable angular expression to data-bind to.
name string string

Property name of the form under which the control is published.
ng-true-value expression expression

The value to which the expression should be set when selected.
ng-false-value expression expression

The value to which the expression should be set when not selected.
ng-change expression expression

Expression to be executed when the model value changes.
md-no-ink boolean boolean

If present, disable ink ripple effects.
aria-label string string

Adds label to checkbox for accessibility. Defaults to checkbox's text. If no default text is found, a warning will be logged.
md-indeterminate expression expression

This determines when the checkbox should be rendered as 'indeterminate'. If a truthy expression or no value is passed in the checkbox renders in the md-indeterminate state. If falsy expression is passed in it just looks like a normal unchecked checkbox. The indeterminate, checked, and unchecked states are mutually exclusive. A box cannot be in any two states at the same time. Adding the 'md-indeterminate' attribute overrides any checked/unchecked rendering logic. When using the 'md-indeterminate' attribute use 'ng-checked' to define rendering logic instead of using 'ng-model'.
ng-checked expression expression

If this expression evaluates as truthy, the 'md-checked' css class is added to the checkbox and it will appear checked.
