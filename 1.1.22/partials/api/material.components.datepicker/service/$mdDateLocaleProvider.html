The $mdDateLocaleProvider is the provider that creates the $mdDateLocale service. This provider that allows the user to specify messages, formatters, and parsers for date internationalization. The $mdDateLocale service itself is consumed by AngularJS Material components that deal with dates (i.e. mdDatepicker).

Usage

myAppModule.config(function($mdDateLocaleProvider) { // Example of a French localization. $mdDateLocaleProvider.months = ['janvier', 'février', 'mars', ...]; $mdDateLocaleProvider.shortMonths = ['janv', 'févr', 'mars', ...]; $mdDateLocaleProvider.days = ['dimanche', 'lundi', 'mardi', ...]; $mdDateLocaleProvider.shortDays = ['Di', 'Lu', 'Ma', ...]; // Can change week display to start on Monday. $mdDateLocaleProvider.firstDayOfWeek = 1; // Optional. $mdDateLocaleProvider.dates = [1, 2, 3, 4, 5, 6, ...]; // Example uses moment.js to parse and format dates. $mdDateLocaleProvider.parseDate = function(dateString) { var m = moment(dateString, 'L', true); return m.isValid() ? m.toDate() : new Date(NaN); }; $mdDateLocaleProvider.formatDate = function(date) { var m = moment(date); return m.isValid() ? m.format('L') : ''; }; // Allow only a day and month to be specified. // This is required if using the 'M/D' format with moment.js. $mdDateLocaleProvider.isDateComplete = function(dateString) { dateString = dateString.trim(); // Look for two chunks of content (either numbers or text) separated by delimiters. var re = /^(([a-zA-Z]{3,}|[0-9]{1,4})([ .,]+|[/-]))([a-zA-Z]{3,}|[0-9]{1,4})/; return re.test(dateString); }; $mdDateLocaleProvider.monthHeaderFormatter = function(date) { return myShortMonths[date.getMonth()] + ' ' + date.getFullYear(); }; // In addition to date display, date components also need localized messages // for aria-labels for screen-reader users. $mdDateLocaleProvider.weekNumberFormatter = function(weekNumber) { return 'Semaine ' + weekNumber; }; $mdDateLocaleProvider.msgCalendar = 'Calendrier'; $mdDateLocaleProvider.msgOpenCalendar = 'Ouvrir le calendrier'; // You can also set when your calendar begins and ends. $mdDateLocaleProvider.firstRenderableDate = new Date(1776, 6, 4); $mdDateLocaleProvider.lastRenderableDate = new Date(2012, 11, 21); });

Properties


  • $mdDateLocaleProvider.months

    Array of month names (in order).

  • $mdDateLocaleProvider.shortMonths

    Array of abbreviated month names.

  • $mdDateLocaleProvider.days

    Array of the days of the week (in order).

  • $mdDateLocaleProvider.shortDays

    Array of abbreviated days of the week.

  • $mdDateLocaleProvider.dates

    Array of dates of the month. Only necessary for locales using a numeral system other than [1, 2, 3...].

  • $mdDateLocaleProvider.firstDayOfWeek

    The first day of the week. Sunday = 0, Monday = 1, etc.

  • $mdDateLocaleProvider.parseDate

    Function that converts a date string to a Date object (the date portion).

  • $mdDateLocaleProvider.formatDate

    Function to format a date object to a string. The datepicker directive also provides the time zone, if it was specified.

  • $mdDateLocaleProvider.monthHeaderFormatter

    Function that returns the label for a month given a date.

  • $mdDateLocaleProvider.monthFormatter

    Function that returns the full name of a month for a given date.

  • $mdDateLocaleProvider.weekNumberFormatter

    Function that returns a label for a week given the week number.

  • $mdDateLocaleProvider.longDateFormatter

    Function that formats a date into a long aria-label that is read by the screen reader when the focused date changes.

  • $mdDateLocaleProvider.msgCalendar

    Translation of the label "Calendar" for the current locale.

  • $mdDateLocaleProvider.msgOpenCalendar

    Translation of the button label "Open calendar" for the current locale.

  • $mdDateLocaleProvider.firstRenderableDate

    The date from which the datepicker calendar will begin rendering. Note that this will be ignored if a minimum date is set. Defaults to January 1st 1880.

  • $mdDateLocaleProvider.lastRenderableDate

    The last date that will be rendered by the datepicker calendar. Note that this will be ignored if a maximum date is set. Defaults to January 1st 2130.

  • $mdDateLocaleProvider.isDateComplete

    Function to determine whether a string makes sense to be parsed to a Date object. Returns true if the date appears to be complete and parsing should occur. By default, this checks for 3 groups of text or numbers separated by delimiters. This means that by default, date strings must include a month, day, and year to be parsed and for the model to be updated.
